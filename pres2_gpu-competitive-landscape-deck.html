FOUNDATIONS
RECAP
1 · The Silicon
From commodity chip to purpose-built AI accelerator
GPU vs CPU
GPUs run thousands of simple parallel operations simultaneously. CPUs run fewer but smarter serial operations. AI workloads are matrix multiplications — pure parallelism.
Hardware form factors
Consumer GPU → data-centre PCIe card → SXM tray (8 GPUs, NVLink) → DGX / HGX rack system. Each step multiplies memory and interconnect bandwidth.
GPU generations
A100 → H100 → H200 → B200 / GB200. Each generation roughly doubles FLOP/s and HBM bandwidth. Blackwell (B200) currently shipping to hyperscalers and top neoclouds.
2 · How AI Runs on GPUs
From matrix multiplications to tokens per second
Token generation loop
LLMs produce one token per forward pass. The K/V cache of prior tokens is stored in HBM and grows with context length — this is why long contexts are expensive.
Memory math
Model memory = layers × width × precision (FP16 / BF16 / INT8). A 70B parameter model in FP16 requires ~140 GB VRAM — already beyond a single H100 (80 GB).
Two bottlenecks
Memory capacity limits how many concurrent users a GPU can serve. HBM bandwidth limits tokens-per-second. Solving both requires multi-GPU serving.
3 · The Economics
From $/GPU-hr to $/token to product unit economics
Cost per token varies 2–5×
Same model, different hardware: H100 PCIe → DGX H100 → DGX B200. NVLink bandwidth and HBM capacity per chip drive efficiency, not just $/GPU-hr headline price.
Training vs inference
Training is a one-time CapEx event (months of compute). Inference runs 24/7 at scale and drives the ongoing product cost base. Inference economics are what this deck is about.
Context window = hidden cost
Doubling context length more than doubles memory cost due to KV cache growth. Long-context products (RAG, agents) are disproportionately sensitive to GPU pricing.

Historical Edge of Orchestrators
Serverless inference endpoints: Pioneered the "scale-to-zero" API model for open-source AI. While hyperscalers required provisioning dedicated instances and paying for idle compute, orchestrators delivered instant, pay-per-token endpoints—radically lowering the barrier to entry for AI developers.
Inference speed innovation: FlashAttention, PagedAttention, and SGLang originated in the orchestrator/research community. These are pieces of 'software shortcuts' that reduced inference cost by 5–24× before hyperscalers built equivalent optimisations.
AI "App Store": First to seamlessly connect to public libraries of free, open-source AI models. Instead of spending weeks wrestling with complicated setup, developers could essentially click a button and launch an AI instantly.
Software-above-GPU category creation: Invented the layer between raw GPU infra and application code (fine-tuning pipelines, serverless inference, experiment tracking) before managed ML services matured.
Pure vs Hybrid Orchestrator
Pure Orchestrators (Modal, Run:ai, W&B): Zero hardware. Scheduling intelligence, experiment tracking, multi-cloud federation. 100% software margin.
Hybrid Orchestrators (Together AI, Lambda): Own GPU capacity AND provide a software platform. Subsidise the software layer with hardware revenue. Together AI maintains GPU clusters with custom inference kernels (FlashAttention maintainers).
Acquired Orchestrators — Market Consolidation
Run:ai (NVIDIA, 2024): Enterprise GPU scheduling. Now embedded in NVIDIA AI Enterprise.
W&B (CoreWeave, 2025): Dominant experiment tracking. Now natively integrated in CoreWeave clusters — XID/AER events feed researcher dashboards directly.

The Fundamental Limit
50ms+ WAN latency = no distributed training (GPU sits idle 99% of the time waiting for gradients). Consumer GPUs lack ECC memory. No SLA, no compliance, no legal recourse. Enterprise data through Akash/Salad risks memory scraping — SOC 2 is structurally impossible for permissionless networks. Ceiling: single-GPU inference and stateless batch jobs tolerant of failure.

Who actually uses Web3 GPU compute?
M · SMEs & Bootstrapped Startups
Salad for Stable Diffusion batches, TTS, and 3D rendering. Akash for censorship-resistant or budget-constrained inference. Workloads are stateless — a dropped node is a minor inconvenience, not a crisis.
Web3-Native Projects
DePIN networks hosting their own RPC nodes, zk-rollup provers, and Web3 front-ends. The biggest customers of Akash and Fluence are other crypto projects — not AI companies.
Who does NOT use this
Fortune 500s, healthcare, finance — bound by GDPR/HIPAA. Proprietary data on Akash/Salad = random machine somewhere with no SLA. They will pay the 3–5× premium to AWS for the legal safety net.
