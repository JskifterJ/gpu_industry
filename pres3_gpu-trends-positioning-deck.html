The Jevons Frame · Foundational Lens
ACT I · THE FORCES
Coined 1865 · Applies exactly to AI compute in 2026
"As AI compute becomes more efficient, total compute consumption grows — not shrinks."
William Stanley Jevons observed in 1865 that improved efficiency of coal-burning engines led to greater, not lesser, coal consumption. The same paradox governs AI infrastructure today. Every algorithmic breakthrough, every hardware efficiency gain, and every price reduction unlocks new demand at a scale that swamps the original saving.
Why this matters: Every trend in this deck is amplified by Jevons. Cheaper inference doesn't reduce GPU demand — it expands it. More efficient training doesn't reduce cluster needs — it enables larger models. Efficiency is the accelerant.
Three proof points
DeepSeek R1 · Jan 2026
Training cost dropped 97% vs GPT-4o. Result: inference API traffic exploded — providers reported 10–40× volume increases within weeks. Cheaper model → dramatically more calls, not fewer.
LLM Inference Cost · Epoch AI
Inference cost per token is halving approximately every 2 months. Industry response: more models deployed, more tokens consumed, more infrastructure required. Price × volume = flat or rising spend.
Algorithmic Efficiency · Epoch AI
Pre-training efficiency improving 3× per year. Each generation trains larger models on the same hardware — the hardware budget isn't reduced, it's reallocated upward. Compute demand keeps growing.
Sources: Epoch AI (epochai.org) · DeepSeek R1 technical report · API volume reports Jan 2026

Sphere I · Hardware · Bandwidth Constraints
HARDWARE
FROM COMPUTE-BOUND
TO MEMORY-BOUND
TFLOPS grow exponentially. Memory bandwidth grows linearly. The gap is widening.
H100 → B200: TFLOPS grew 4.5×. Memory bandwidth grew 2.4×. The gap is widening each generation. When BW growth < TFLOPS growth → more workloads become memory-bound.
↓ Why this matters for GPU provider selection
Providers advertising raw TFLOPS are often misleading. For LLM inference, effective throughput is memory-bandwidth-limited. Key questions: HBM bandwidth per GPU? NVLink/NVMe topology?
MIG on Blackwell 6000 preserves HBM bandwidth allocation per instance — crucial for inference quality at partition boundaries.
The Memory Wall
70B BF16: 140 GB model weights. KV cache adds 10–260 GB depending on context length. Memory is the dominant resource constraint — not raw TFLOPS. At B200: a 70B inference run is ~85% memory-bandwidth-limited.
Compute → Memory Transition
Inference on H100 for Llama 3.1 70B: ~85% of time is spent waiting for weight transfers from HBM — not computing. Adding TFLOPS alone doesn't help. The GPU is idle, waiting for memory.
Interconnect: Decisive, Not New
NVLink 5.0: 1,800 GB/s. PCIe 5.0: 64 GB/s. 28× gap for multi-GPU jobs. The memory wall is a decades-old HPC concept — what's changed is its relative severity as TFLOPS outpace bandwidth.

THE INFERENCE EFFICIENCY RACE
Inference cost dropped ~280× in 2 years. The software layer drove most of it. The race is now about who captures that margin — and in what new workload shapes.
vLLM / SGLang / TRT-LLM
PagedAttention (vLLM): 2–4× throughput vs naïve batching. SGLang's RadixAttention: cross-request KV cache reuse. NVIDIA TRT-LLM: hardware-optimised engine. The open-source inference stack is approaching proprietary performance.
FlashAttention + Speculative Decoding
FlashAttention-3: up to 75% GPU utilisation on H100 (vs ~35% without). Speculative decoding (draft + verify): 2–3× tokens/sec for long generation tasks. Kernel-level optimisation now compounds with serving-level gains.
MIG — Multi-Instance GPU
Blackwell 6000: up to 7 isolated instances per GPU with dedicated HBM bandwidth. Inference workloads ≤13B parameters use <1/7th of a GPU — MIG lets you sell 7 inference slices from one card. Critical for inference-heavy clouds.
Cost Trajectory
GPT-4-class inference: ~$60/1M tokens (Jun 2023) → ~$0.30/1M tokens (Apr 2026). 200× reduction. MoE architecture (Gemma 4 26B A4B): inference cost now tracks active params — 4B active of 26B total. MoE repriced the inference market.
L40S / A40 as Inference Tier
Not every inference workload needs H100. L40S: 48 GB GDDR6 at $7–8K/card. Cost-optimised tier for <70B inference. Hardware stratification is happening — inference ≠ training hardware. ICN's Blackwell mix allows tiering.
Managed Agents (Anthropic, Apr 2026)
Decouples the brain (Claude + tools) from the hands (sandbox containers) — provisioned on-demand only when compute is needed. p50 latency −60%, p95 −90%. Session runtime: $0.08/session-hr. New workload shape: sustained sessions vs. burst requests.
