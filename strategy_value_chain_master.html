Business Model Archetypes

7 distinct strategies for capturing value in the GPU compute stack. Click any node on the canvas to see how each model relates to that component.

H · Hyperscaler
AWS · GCP · Azure · Oracle
Vertically integrated from silicon to SaaS. Own custom AI chips (TPU v5, Trainium2, Inferentia2), global data center networks (100+ regions), CUDA-equivalent software stacks, and end-user AI products. The hyperscaler bundles compliance, managed services, and enterprise SLAs into a single vendor relationship — the "default choice" for enterprises that want to avoid procurement complexity.
Layers: L0–L7 full-stack ownership
Moat: Breadth of platform — 200+ managed services, global compliance certs (FedRAMP, HIPAA, ISO 27001), and the stickiness of multi-service integration. Custom silicon creates cost advantages at scale that neoclouds cannot replicate.
Risk: GPU allocation rationed — hyperscalers don't always prioritise AI inference/training capacity. Premium pricing (30–50% above neocloud equivalents). Slower innovation cycle than specialist providers.
AWS (Bedrock, SageMaker, UltraClusters) · GCP (Vertex AI, TPU, A3 Mega) · Azure (OpenAI Service, Azure ML, ND-series) · Oracle (OCI Supercluster, Stargate)
N · Neocloud
CoreWeave · Nebius · Lambda · Crusoe · Fluidstack
GPU-first clouds that own the hardware but not the software platform. Lease colocation space and power, procure GPUs at scale (GPU procurement is existential — access to H100/B200 allocation determines revenue growth), and provide raw compute as bare metal or VMs. Compete on cluster quality (InfiniBand fabric, GPU health checks), pricing (15–30% below hyperscalers), and speed of provisioning.
Layers: L0 (procurement) · L1 (infra) · L2 (drivers) · L3 (orchestration) · L5 (partial storage)
Moat: GPU procurement relationships and cluster quality (InfiniBand fabric integrity). CoreWeave commands 10–15% premium over peers due to SUNK Slurm-on-K8s quality. Long-term customer commitments (12–36 months) create revenue visibility.
Risk: Margin compression as GPU supply normalises. Capital-intensive model — $1B+ debt to fund GPU procurement. Customer concentration (CoreWeave: OpenAI = 62% of revenue at IPO). Blackwell transition timing risk.
CoreWeave (CRWV, $1.9B rev, Platinum ClusterMAX) · Nebius ($NBIS, Gold) · Crusoe (Stargate JV) · Lambda · Fluidstack (Gold, Google-backed)
A · Aggregator
Vast.ai · TensorDock · Lepton (NVIDIA) · spot markets
Zero hardware ownership. Operates a marketplace or unified API layer that routes compute requests to the cheapest available provider in real time. The value is arbitrage and convenience — the aggregator charges a spread between what capacity providers offer and what customers are willing to pay. Structurally excluded from training (requires dedicated IB fabric that aggregators can't guarantee).
Layers: L3 (federation) · L6 (marketplace/billing) · L7 (unified API)
Moat: Liquidity (more supply → better prices → more demand → more supply). Network effects in the marketplace. API unification reduces switching cost from any single provider to near-zero.
Risk: Thin margins (spread compression as market matures). Cannot serve latency-sensitive or compliance-heavy customers. Disintermediation if providers go direct. Spot-only model means unreliable SLAs.
Vast.ai (P2P GPU marketplace) · TensorDock · io.net (DePIN adjacent) · NVIDIA Lepton (compute marketplace)
O · Orchestrator / ML Platform
Run:ai · Modal · Together AI · SambaNova · Weights & Biases
Software-only or software-first. Does not own GPUs — sits above hardware providers and adds scheduling intelligence, ML workflow management, and developer experience. Run:ai maximises GPU utilisation via Kubernetes-native scheduling. Modal abstracts infrastructure entirely — deploy a Python function, get H100 access in seconds. Together AI hosts open-source models and provides fine-tuning APIs.
Layers: L3 (scheduling) · L4 (execution) · L5 (MLOps) · L6 (observability) · L7 (developer API)
Moat: Developer experience and workflow lock-in. Run:ai (acquired by NVIDIA $700M 2024) embeds deep into enterprise GPU fleets. MLOps tooling (experiment tracking, model registry) creates high switching costs once adopted.
Risk: Hyperscaler encirclement — AWS SageMaker, Azure ML, Vertex AI all compete directly. Hardware-agnostic positioning is a selling point but also prevents owning the full margin stack.
Run:ai (NVIDIA, $700M acq.) · Modal (serverless GPU) · Together AI (Silver ClusterMAX) · SambaNova (custom RDU silicon) · Fireworks AI
D · DePIN
io.net · Akash Network · Nosana
Decentralised Physical Infrastructure Networks. Aggregate idle GPU capacity from individuals and small operators, coordinate via blockchain smart contracts, and offer compute at significant discounts (60–80% below hyperscalers). Economically viable only for workloads that tolerate latency variability and don't require InfiniBand — primarily inference and LoRA/QLoRA fine-tuning on consumer-grade GPUs (RTX 3090/4090).
Layers: L1 (peer nodes, variable quality) · L3 (blockchain-based orchestration) · L4 (inference/FT only)
Moat: Price — structurally lowest cost when workload fits. Token incentive model can attract GPU supply ahead of demand. Community and Web3 ecosystem alignment.
Risk: Structurally excluded from training (IB required). Quality variability makes SLAs impossible. Security posture of peer nodes unverifiable — non-starter for regulated industries. Crypto payment friction for enterprise buyers.
io.net (Solana, $30M Series A, inference clusters) · Akash Network (Cosmos SDK, open-source) · Nosana (Solana, CI/CD GPU)
B · Hybrid Enterprise
On-prem GPU cluster + cloud burst
Enterprise IT model where the organisation owns on-premises GPU hardware (typically DGX H100 servers or OEM equivalents) and uses cloud (hyperscaler or neocloud) for burst capacity. Data sovereignty is the primary driver — the training corpus or fine-tuning dataset cannot leave the premises. The on-prem cluster becomes the primary IP-generation engine; cloud handles overflow and inference scale-out.
Layers: L0 (OEM procurement) · L1 (on-prem DC) · L2–L7 (self-managed or outsourced)
Moat: Data sovereignty — proprietary training data and model weights never leave secure perimeter. CapEx model amortises over 4–5 years, creating long-term cost advantages vs OpEx-only cloud. Full hardware control enables custom optimisation.
Risk: GPU procurement timing (6–18 month lead times). Cooling/power upgrades to on-prem DCs for H100 racks. Stranded capacity during model release cycles. MLOps staffing burden.
Financial institutions · Healthcare AI · Government labs · Any enterprise with material data-sovereignty requirements
I · Inference API / Model Provider
OpenAI · Anthropic · Google DeepMind · Mistral
Own the highest-value layer: the model weights. Training pipelines (RLHF, Constitutional AI, FSDP) and inference infrastructure (vLLM, custom caching, batching) are both owned. Inference IS the revenue product — ChatGPT, Claude.ai, Gemini serve hundreds of millions of daily requests. Depend on hyperscalers for L0–L3 but own L4–L7 fully, including consumer applications.
Layers: L1–L3 (leased/partnered) · L4 Training + Inference (owned) · L5 Data pipelines (owned) · L6 (owned) · L7 Consumer apps (owned)
Moat: Model quality (benchmark leadership → developer adoption → revenue → more training compute → better model). RLHF/safety fine-tuning data is proprietary and hard to replicate. Consumer brand (ChatGPT, Claude) creates direct distribution.
Risk: Existential dependency on hyperscaler infrastructure (Anthropic on AWS/GCP, OpenAI on Azure/CoreWeave). Training compute cost spiral. Open-source models (Llama, Mistral) commoditising the model layer from below.
OpenAI (GPT-4o, Azure infra) · Anthropic (Claude, AWS/GCP) · Google DeepMind (Gemini, TPU) · Mistral (Le Chat, EU sovereign)

Stack Layer Guide

8 layers from raw silicon to end-user apps. Each colored band on the canvas = one layer. Click any node for full detail.

L0 · Silicon & Supply Chain
The chips themselves. NVIDIA H100/B200 (dominant), AMD MI300X, custom ASICs (Google TPU v5, AWS Trainium2, Amazon Inferentia2). TSMC 4nm/3nm foundry, HBM3e memory stacks, CoWoS advanced packaging. Export controls (BIS Entity List) limit which countries can receive H100-class chips — a geopolitical constraint that shapes the entire market.
Lead time: 6–18 months · Hyperscaler custom ASICs exempt from NVIDIA dependency
L1 · Physical Infrastructure
Data centers (owned or leased colocation), utility power (40–500MW per campus), liquid cooling (direct liquid cooling mandatory for H100 — air cooling insufficient), GPU server racks (8× H100 SXM5 per HGX node), and the InfiniBand fabric (400G NDR) that connects nodes for distributed training. The IB fabric is the critical infrastructure that DePIN cannot provide — it's why multi-node training requires a dedicated neocloud or hyperscaler.
1× H100 node = ~10kW · 1,000-node cluster = 15–20MW continuous · Power = primary site selection driver
L2 · Firmware, Drivers & Compute Libraries
CUDA is NVIDIA's GPU programming language — the dominant standard, equivalent to Windows for PCs. If your code runs CUDA, it runs on NVIDIA GPUs (96% market share in AI training). NCCL ("Nickel") is the communication library that coordinates thousands of GPUs during distributed training — like a conductor keeping an orchestra in sync. PyTorch / JAX are the frameworks AI researchers write models in; they sit above CUDA and abstract away raw GPU programming.
CUDA lock-in = NVIDIA moat · ROCm (AMD) improving but 3–5 years behind · XLA (Google TPU equivalent)
L3 · Virtualisation & Orchestration
Kubernetes is the industry-standard "traffic controller" that decides which AI job gets which GPUs at what time — like an air traffic controller for compute. Slurm is the older HPC scheduler used in supercomputers; increasingly wrapped in Kubernetes (Slurm-on-Kubernetes = SUNK at CoreWeave). MIG partitioning lets one H100 act as 7 independent smaller GPUs — useful for small inference workloads. Capacity aggregation is the Aggregator model's core L3 play: routing requests across multiple providers via a unified API.
GPU utilisation target: 80–90% · Below 60% = neocloud economics break down · MIG: 1×H100 → 7×10GB instances
L4 · Workload Execution (Training · Fine-Tuning · Inference)
Three parallel channels with different hardware requirements. Training: weeks/months, thousands of H100s, IB fabric mandatory, FSDP/Megatron-LM for parallelism, gradient checkpointing for VRAM. Fine-tuning: hours/days, LoRA/QLoRA reduces VRAM 4–8×, consumer RTX 4090 viable (DePIN sweet spot), RLHF/DPO for alignment. Inference: continuous 24/7 serving, vLLM continuous batching (10× throughput vs naïve), INT8/FP8 quantisation, speculative decoding — THIS is where revenue is generated.
Training: $1–100M per run · Fine-tune: $10–100k · Inference: continuous OpEx · KV cache = memory bottleneck
L5 · Data Pipelines & Serving Infrastructure
Training data pipelines: petabyte-scale S3 reads, WebDataset/FFCV async prefetch, tokenisation, MinHash LSH deduplication, quality filtering — a data pipeline bottleneck can cap GPU utilisation below 40%. Fine-tune data: curated JSONL instruction pairs, RLHF preference datasets (Scale AI, LabelBox), synthetic data generation (Self-Instruct, Magpie). Inference serving: vLLM/Triton endpoints, API gateways, vector databases for RAG (Pinecone, Weaviate, pgvector), semantic caching to eliminate redundant GPU compute.
S3 throughput: GB/s sustained for training · Vector DB query latency: <50ms for RAG · Cache hit rate target: 30–60%
L6 · Control Plane · FinOps · Security
Control plane: model router (which model serves which request), multi-model A/B testing, dynamic routing by cost/latency/capability. FinOps: GPU-hour cost attribution per team/project, chargeback models, reserved vs spot optimisation — at $2–3/hr per H100, cost visibility is a $10M+ annual decision. Security: IAM + RBAC (who can access which model), data privacy (PII redaction in prompts), compliance certs (SOC2 Type II, HIPAA, FedRAMP) that are enterprise procurement gates.
SOC2 Type II = minimum enterprise gate · FedRAMP = US federal requirement · HIPAA = healthcare AI · GDPR = EU compliance
L7 · Application & End-User Interface
The consumer-facing layer where AI value is delivered. RAG pipelines (Retrieval-Augmented Generation): query a vector DB at inference time to ground LLM responses in current/private data without retraining. Agents & tool calling: LLMs that can browse the web, execute code, call APIs, and chain multi-step reasoning. Workflow automation: LangChain/LlamaIndex orchestration. Consumer apps: ChatGPT, Claude.ai, Gemini — the revenue-generating surfaces. Enterprise integration: Slack/Teams bots, Salesforce AI, document processing pipelines.
RAG: eliminates hallucination on proprietary data · Agents: 5–10× inference token cost vs single-turn · L7 = where BMs diverge most
★ S3 / Object Storage nodes — amber left border on canvas nodes = high-throughput object storage integration point. Training checkpoints, model registry, dataset reads. Primary infrastructure wedge for storage-adjacent plays.
