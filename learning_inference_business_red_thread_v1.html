Chapter 4 - Model sizing at high level

Model Size, Precision, and Context Requirements

For executive planning, three inputs are usually enough: parameters, precision, and context length.

Input 1

Model parameters

Bigger models generally need more memory and more distributed execution. This pushes you toward stronger interconnects and scheduler discipline.

Input 2

Precision / quantization

Lower precision reduces memory pressure and can raise throughput, but may impact quality for some workloads. It is a business trade-off, not just a technical tweak.

Input 3

Context and output length

Long prompts and long outputs inflate memory traffic and queue times. This directly shapes user-perceived latency and cost per response.

Chapter 9 - Storage lifecycle map

Storage and Retention Architecture

Storage shape directly influences latency, compliance posture, and long-term cost.

Hot tier (ms-seconds)
Online vector + cache

RAG retrieval fragments, session memory, active feature stores.

Model-serving cache

Frequently loaded model artifacts and tokenizer assets.

Warm tier (seconds-minutes)
Object/block storage

Recent prompts, outputs, and application traces for replay and QA.

Operational logs

Usage, billing events, and policy decisions for governance.

Cold tier (hours-days+)
Archive (on-prem or off-prem)

Compliance retention, legal hold, model lineage snapshots.

Analytics lake

Long-horizon demand and quality analysis for roadmap and pricing.

Process reminder: context retrieval path and output retention path should be explicitly shown in architecture reviews. Hidden storage assumptions become hidden costs.

Chapter 12 - Close and action plan

Decision Sequence and Operating Plan

Keep the red thread intact from architecture to commercial outcomes.

Step 1

Segment workload demand

Build demand tiers by latency, context size, and concurrency pattern.

Step 2

Size model and precision

Select quality envelope first, then fit and batch for practical throughput.

Step 3

Design for MFU stability

Batch policy + interconnect + scheduling should be treated as one system.

Step 4

Embed isolation and governance

Trust and compliance controls are mandatory for enterprise scale.

Step 5

Benchmark economics in context

Measure cost by delivered output quality and SLA, not by token list price alone.

Step 6

Operationalize as a living loop

Continuously recalibrate workload mix, model routing, and capacity allocation.

The platform with the strongest red thread from workload to MFU to economics will outperform in both margin and customer experience.
