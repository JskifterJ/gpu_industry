Chapter 1 - Executive framing

From GPU Hardware to Business Outcome: One Red Thread

We align technical decisions to business value by tracking one throughline: workload shape -> system bottlenecks -> MFU -> cost and user experience.

Audience: Mixed business + technical decision makers Focus: Inference first, with links to model customization and training paths Core metric: MFU (Model FLOPs Utilization)
What changed from v10

More business-relatable, less model math

Chat is one workload, not the workload. We include image, speech, RAG, and agents.
Model internals are compressed to only what drives infrastructure and cost.
Model fit + concurrency planning are now one combined decision panel.
Security and multi-tenant isolation are explicit in the architecture, not assumed.
Guiding question

What are we really buying?

You are not buying only GPUs. You are buying reliably low-latency output tokens under realistic concurrency, with safe multi-tenant isolation and predictable economics across mixed workloads.

Decision quality improves when every architecture conversation ends at the same scoreboard: user latency, throughput, MFU, and cost per output token.

Chapter 4 - Model sizing at high level

Only Three Inputs Determine Most Hosting Requirements

Parameters, precision, and context length. This is enough for executive-level infra planning before deep model math.

Input 1

Model parameters

Bigger models generally need more memory and more distributed execution. This pushes you toward stronger interconnects and scheduler discipline.

Input 2

Precision / quantization

Lower precision reduces memory pressure and can raise throughput, but may impact quality for some workloads. It is a business trade-off, not just a technical tweak.

Input 3

Context and output length

Long prompts and long outputs inflate memory traffic and queue times. This directly shapes user-perceived latency and cost per response.

Chapter 8 - Security, isolation, and governance

Multi-Tenant Trust Is a Product Requirement

For buyers, this is not “extra architecture.” It is part of core service quality and procurement confidence.

Isolation plane

Compute boundaries

Per-tenant quotas, GPU partitioning, network policy, and namespace-level controls prevent noisy neighbors and data crossover.

Data plane

Retention + residency

Control plane

Auditability

Chapter 9 - Storage lifecycle map

Where Context Comes From and Where Outputs Go

Storage is not a side note. It shapes latency, compliance posture, and long-term operating cost.

Hot tier (ms-seconds)
Online vector + cache

RAG retrieval fragments, session memory, active feature stores.

Model-serving cache

Frequently loaded model artifacts and tokenizer assets.

Warm tier (seconds-minutes)
Object/block storage

Recent prompts, outputs, and application traces for replay and QA.

Operational logs

Usage, billing events, and policy decisions for governance.

Cold tier (hours-days+)
Archive (on-prem or off-prem)

Compliance retention, legal hold, model lineage snapshots.

Analytics lake

Long-horizon demand and quality analysis for roadmap and pricing.

Process reminder: context retrieval path and output retention path should be explicitly shown in architecture reviews. Hidden storage assumptions become hidden costs.

Chapter 12 - Close and action plan

Decision Sequence for High-Confidence Inference Strategy

Keep the red thread intact from architecture to commercial outcomes.

Step 1

Segment workload demand

Build demand tiers by latency, context size, and concurrency pattern.

Step 2

Size model and precision

Select quality envelope first, then fit and batch for practical throughput.

Step 3

Design for MFU stability

Batch policy + interconnect + scheduling should be treated as one system.

Step 4

Embed isolation and governance

Trust and compliance controls are mandatory for enterprise scale.

Step 5

Benchmark economics in context

Measure cost by delivered output quality and SLA, not by token list price alone.

Step 6

Operationalize as a living loop

Continuously recalibrate workload mix, model routing, and capacity allocation.

Final message: the platform with the strongest red thread from workload to MFU to economics will outperform in both margin and customer experience.
